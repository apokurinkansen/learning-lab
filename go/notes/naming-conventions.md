# 命名規則

## 概要
Go の命名規則はパスカルケース/キャメルケースを使い、先頭の大文字/小文字がアクセス制御の仕組みになっている。

## 学んだこと
- 先頭が大文字（パスカルケース）→ エクスポート（公開）: `Println`, `NewReader`
- 先頭が小文字（キャメルケース）→ 非エクスポート（非公開）: `myFunc`, `count`
- スネークケース（`_`）は使わない

### なぜスネークケースを使わないか
- 先頭の大文字/小文字によるアクセス制御との相性が悪い
- `do_something` vs `Do_something` は読みにくい
- 他の言語の `public` / `private` キーワードの代わりに大文字/小文字を使うという設計判断

## コード例

```go
func DoSomething() {} // 他のパッケージから呼べる（公開）
func doSomething() {} // このパッケージ内でしか呼べない（非公開）
```
