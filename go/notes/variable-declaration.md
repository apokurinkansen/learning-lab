# 変数宣言

## 概要
Go には `var` による明示的宣言と `:=` による短縮宣言の2種類がある。関数内では `:=` が主流。

## 学んだこと

### 2つの宣言方法
- `var x int = 10` — 型を明示して宣言
- `x := 10` — 型を省略して宣言（コンパイラが推論する）
- どちらも「新しい変数を作る」という宣言。`:=` は型を省略しているだけで、宣言を省略しているわけではない

### なぜ `:=` と `=` を分けるのか
- `=` だけだと宣言なのか代入なのかコンパイラが区別できない
- タイポで意図しない変数が生まれるのを防げる（Python のような言語では検出できない）

### 使い分け
| 書き方 | 使う場面 |
|---|---|
| `x := 10` | 関数内で、型が明らかなとき（普段はこっち） |
| `var x int` | 初期値なしで宣言したいとき（ゼロ値が入る） |
| `var x int = 10` | 型を明示したいとき |

### `:=` が使えない場面
- 関数の外（パッケージレベル）では `var` を使う必要がある

## コード例

```go
// 関数内では := が主流
name := "Go"
count := 42
result, err := fetchData()

// var を使う場面
var version = "1.0"       // パッケージレベル
var count int              // ゼロ値で初期化
var x float64 = 10         // 型を明示（:= だと int になる）
```
